# Implementation Plan: Core Binary with CLI and REST Daemon

**Branch**: `001-core-binary-cli` | **Date**: 2026-03-31 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-core-binary-cli/spec.md`

## Summary

Single Rust binary (`csn`) that loads ONNX embedding models via fastembed-rs, classifies text against configurable TOML reference sets, and serves both CLI and REST interfaces. The daemon hot-reloads reference sets via file watching. This is a refactor of the existing prototype code to align with spec and constitution.

## Technical Context

**Language/Version**: Rust 2024 edition (1.85+)
**Primary Dependencies**: fastembed 5.13, axum 0.8, clap 4, notify 7, tokio 1
**Storage**: Filesystem — TOML config, TOML reference sets, binary embedding cache (blake3-hashed)
**Testing**: cargo test (unit + integration)
**Target Platform**: macOS (Apple Silicon), Linux x86_64
**Project Type**: CLI + daemon (single binary)
**Performance Goals**: <50ms p95 warm classification, <3s daemon startup
**Constraints**: localhost-only, single binary, no runtime dependencies
**Scale/Scope**: 2-12 reference sets, 5-100 phrases each, single concurrent user
**Distribution**: crates.io (`cargo install computer-says-no`) + GitHub release binaries

**Structure Decision**: Flat single-crate layout. No workspace — the binary is the only deliverable. Follows astro-up's "thin CLI, thick core" principle but without the workspace split since we have one frontend. Modules map 1:1 to spec entities.

### Key Refactors from Prototype

1. **config.rs** (NEW): The prototype has no config module — port/model/paths are hardcoded or CLI-only. Need 3-layer config precedence per FR-012.
2. **model.rs**: Change `EmbeddingEngine` to not store `model` field redundantly. Fix `embed()` to work with shared state pattern.
3. **server.rs**: Replace `Vec<ReferenceSet>` with `RwLock<Vec<ReferenceSet>>` for atomic swap on hot-reload. Add proper error responses (JSON with error field, correct HTTP status codes). Add graceful shutdown.
4. **watcher.rs** (NEW): `notify::RecommendedWatcher` watching sets directory, debounced, triggers re-embed + atomic swap.
5. **main.rs**: Add config loading. CLI classify/embed/similarity become thin HTTP clients to daemon (fail if daemon down per clarification). `csn serve` wires up watcher + server.
6. **reference_set.rs**: Minor — ensure `load_all_reference_sets` can be called for reload (already works).
