# Implementation Plan: Model Benchmark Harness

**Branch**: `002-model-benchmark-harness` | **Date**: 2026-03-31 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/002-model-benchmark-harness/spec.md`

## Summary

Add `csn benchmark` subcommand that compares all 12 embedding models across LLM-generated labeled datasets, measuring warm classification latency (p50/p95/p99), cold startup time, and accuracy. Includes `csn benchmark generate-datasets` for LLM-based dataset creation with parallel generation per reference set.

## Technical Context

**Language/Version**: Rust 2024 edition (1.85+)
**Primary Dependencies**: fastembed 5.13 (existing), clap 4 (existing), serde_json (existing), indicatif (progress — already transitive dep of fastembed), comfy-table (NEW — table formatting)
**Storage**: JSON files in `datasets/` directory, JSON output for results
**Testing**: cargo test (unit + integration)
**Target Platform**: macOS (Apple Silicon), Linux x86_64
**Project Type**: CLI extension to existing `csn` binary
**Performance Goals**: Full 12-model benchmark within 10 minutes, stable warm latency (CV <30%)
**Constraints**: Single binary (no external benchmark tools), minimal new deps per Principle V
**Scale/Scope**: 12 models × 2-4 datasets × 50 prompts × 20 iterations = ~24,000-48,000 classifications per full run

**Structure Decision**: Two new flat modules following existing pattern. `benchmark.rs` handles the measurement loop and reporting. `dataset.rs` handles dataset schema, loading, and generation. No subfolders — Principle V.

### Key Design Decisions

1. **Inline percentile calculation**: Sort latencies, index at positions `len * 0.50`, `len * 0.95`, `len * 0.99`. No crate needed for 3 lines of math.
2. **indicatif for progress**: Already a transitive dependency of fastembed — zero cost to add as direct dep.
3. **Dataset generation**: Datasets are JSON files generated externally (by LLM via subagents during speckit implementation, not at runtime). The `generate-datasets` subcommand orchestrates this but the actual prompts come from LLM interaction, not from Rust code. For the binary, `generate-datasets` reads reference sets and outputs a template that can be filled by an LLM, or accepts pre-generated files.
4. **No new test framework**: Use `cargo test` with the generated datasets as test fixtures. The benchmark command itself is the integration test.
