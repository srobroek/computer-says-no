# Implementation Plan: MLP Classifier

**Branch**: `003-mlp-classifier` | **Date**: 2026-04-01 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/003-mlp-classifier/spec.md`

## Summary

Add a combined classification pipeline (embedding + cosine features → 2-layer MLP) that automatically replaces pure cosine similarity for binary reference sets. Trains at startup from reference set phrases using the Burn framework (NdArray backend), caches weights via blake3-keyed MessagePack files. No API changes — improvement is transparent to clients. Target: 89-96% accuracy vs ~80% cosine baseline.

## Technical Context

**Language/Version**: Rust 2024 edition (1.92)

## Implementation Approach

### Phase 1: Foundation (src/mlp.rs)

1. Add `burn` and `burn-ndarray` to Cargo.toml
2. Define `MlpClassifier` module using Burn's `#[derive(Module)]`
3. Define `MlpConfig` using Burn's `#[derive(Config)]`
4. Implement forward pass: 387→256(ReLU)→128(ReLU)→1(sigmoid)
5. Implement training loop: custom loop with Adam, L2 (weight decay), early stopping
6. Implement weight save/load using `NamedMpkFileRecorder` + blake3 content hash

### Phase 2: Integration (classifier.rs, config.rs)

7. Add `[mlp]` config section to `AppConfig` (fallback, hyperparameters)
8. Create combined pipeline function in classifier.rs:
   - Compute cosine features (max_pos, max_neg, margin) from existing `best_match`
   - Concatenate with raw embedding → 387-dim tensor
   - Forward through MLP → sigmoid → confidence
   - Build `BinaryResult` with MLP confidence + cosine scores
9. Modify `classify()` to use combined pipeline when `TrainedModel` is available

### Phase 3: Startup & Cache (main.rs, server.rs)

10. Train MLP at daemon startup after reference sets load
11. Store `TrainedModel` in `AppState` (behind `RwLock` like reference sets)
12. Implement cache check: load from `.mpk` if content hash matches, else train
13. Handle training failure: error or fallback based on config

### Phase 4: Hot-Reload (watcher.rs)

14. Extend file watcher callback to retrain MLP when reference set changes
15. Swap model atomically (old model serves requests during retrain)

### Phase 5: Benchmark (benchmark.rs, main.rs)

16. Add "combined" strategy to `compare-strategies`
17. Benchmark combined pipeline accuracy vs cosine on all datasets

### Phase 6: Testing

18. Unit tests for MLP forward pass, training convergence, weight round-trip
19. Integration test: classify via REST with combined pipeline
20. Test cache invalidation (modify set → retrain)
21. Test fallback behavior (no negatives, too few phrases, config override)

## Key Design Decisions

See [research.md](research.md) for full details:

- **Burn over hand-rolled**: Autodiff + Adam + weight serialization for free vs 500+ LOC of manual backprop
- **Combined input (387-dim)**: 96.2% accuracy vs 89.5% embeddings-only
- **MessagePack cache**: Native Burn format, fast load, blake3-keyed invalidation
- **Error-by-default on training failure**: Fail loud, override via config
- **No API changes**: Combined pipeline is transparent to existing clients
