# Implementation Plan: Lazy Auto-Starting Daemon

**Branch**: `005-lazy-daemon` | **Date**: 2026-04-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-lazy-daemon/spec.md`

## Summary

Add transparent lazy daemon to csn CLI. When classify/embed/similarity is invoked, the binary checks for a running daemon via unix socket. If warm: route request over socket (~5ms). If cold: fork a detached daemon, wait for socket, then route (~254ms once). Daemon self-exits after idle timeout. No user-facing daemon management required.

## Technical Context

**Language/Version**: Rust 2024 edition (1.92)
**Primary Dependencies**: tokio (existing, via rust-mcp-sdk), nix 0.29 (new, for PID alive checks), serde_json (existing)
**Storage**: Unix socket + PID file + lock file in `~/.cache/computer-says-no/`
**Testing**: cargo test (unit + integration)
**Target Platform**: macOS (primary), Linux (secondary)
**Project Type**: CLI tool with background daemon
**Performance Goals**: <30ms warm path, <500ms cold path
**Constraints**: Single binary, no new heavy dependencies
**Scale/Scope**: Single user, single machine, ~5 requests/minute typical
