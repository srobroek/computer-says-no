# Implementation Plan: MLP Multi-Category Classification

**Branch**: `008-mlp-multi-category` | **Date**: 2026-04-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/008-mlp-multi-category/spec.md`

## Summary

Extend the MLP classifier from binary (sigmoid/BCE) to multi-category (softmax/cross-entropy). The multi-category MLP produces per-category confidence scores, enabling the system to distinguish correction, frustration, and neutral signals in a single classify call. The corrections.toml reference set is restructured from binary to multi-category. Hook integration reads category from classify output.

## Technical Context

**Language/Version**: Rust 2024 edition (1.92)
**Primary Dependencies**: burn 0.20 (ndarray + autodiff + train), fastembed 5.13, clap 4, rust-mcp-sdk 0.9
**Storage**: TOML reference sets, binary MLP weight cache (`.mpk`), binary embedding cache (blake3)
**Testing**: cargo test (55 unit tests + integration tests)
**Target Platform**: macOS (darwin), Linux
**Project Type**: CLI + MCP server + daemon
**Performance Goals**: <15ms warm classify, <2x binary MLP latency
**Constraints**: Single binary, no network ports, unix socket only
**Scale/Scope**: ~1600 phrases in corrections.toml, 3-4 categories
